rf_classifier Random Forest Model Release Notes

March 20, 2020 v1.2

Main change was to implemented new models that exclusively use autoregression to reduce feature extraction time. Other updates include:

  • New models trained on considerably more data - ~2000 examples from Spring 2019 - Winter 2020
  • 0L04 excluded from training and prediction based on guidance from Tom Powers
  • Removal of tsfresh as a dependency
  • statsmodel package is locked to v0.9 RFFC-13
  • Fixed bug in setup-certified.bash compact RFFC-11

January 9, 2020 v1.1

Various updates made to support retrained models using improved training process.

  • Models saved in a non-pickly format. Hopefully this will help with architecture (32 vs 64 bit) issues
  • Features extracted by tsfresh are now standardized (t-scores based on training data population).
  • Documentation updated to include to new jupiter notebook, pip list output, and test set
  • Updated requirements.txt file

Updated methods from analyze() on down to allow caller to specify which MYA deployment to use. Needed for testing.

Updated certified install instructions (in README.MD) to properly handle virtual environment absolute path problems.

Root directory includes updated requirements.txt and requirements-certified.txt files which only specify directly imported packages. New requirements-dev.txt file captures full environment at development time.

Fixed issue where supplying no event directory generated an exception. Now it prints an error/usage message.

requirements.txt only specifies modules imported by app and not everything exported by pip freeze.

requirements-certified removed since requirements.txt is now identical.
