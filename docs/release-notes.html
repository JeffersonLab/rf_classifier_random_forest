rf_classifier Random Forest Model Release Notes

December 2, 2019 v1.1

Various updates made to support retrained models using improved training process.

  • Models saved in a non-pickly format. Hopefully this will help with architecture (32 vs 64 bit) issues
  • Features extracted by tsfresh are now standardized (t-scores based on training data population).
  • Documentation updated to include to new jupiter notebook, pip list output, and test set info
  • New requirements.txt file only specifies modules imported by app and not everything exported by pip freeze

Updated methods from analyze() on down to allow caller to specify which MYA deployment to use. Needed for testing

Updated certified install instructions (in README.MD) to update the VIRTUAL_ENV variable after copying to certified location

Root directory includes updates requirements.txt, and requirements-certified.txt files which only specify directly imported packages. New requirements-dev.txt file captures full environment at development time.
